# 🍕 Restaurant RAG Assistant

A Retrieval-Augmented Generation (RAG) chatbot that answers questions about a restaurant using only its own documents — menu, customer reviews, and promotional info. Built with LangChain, Chroma, and Ollama running locally.

---

## What It Does

Instead of relying on general LLM knowledge, the assistant retrieves relevant chunks from the restaurant's actual documents before generating an answer. If the answer isn't in the documents, it says "I don't know" — no hallucination.

**Supported document types:**
- CSV — customer reviews
- PDF — restaurant menu
- Images — promotional info (e.g. student discount)

---

## Tech Stack

| Component | Tool |
|-----------|------|
| LLM | Llama 3.2 via [Ollama](https://ollama.com) (runs locally) |
| Embeddings | `all-minilm` via Ollama |
| Vector Database | [Chroma](https://www.trychroma.com/) |
| RAG Framework | [LangChain](https://www.langchain.com/) |
| UI | [Streamlit](https://streamlit.io/) |
| PDF Parsing | pypdf |
| Image Text | Manual extraction (Tesseract OCR support planned) |

## How to Run

### 1. Install Ollama and pull the required models

```bash
# Install Ollama: https://ollama.com/download
ollama pull llama3.2
ollama pull all-minilm
```

### 2. Clone the repo

```bash
git clone https://github.com/OmarADev/restaurant-rag-assistant.git
cd restaurant-rag-assistant
```

### 3. Create a virtual environment and install dependencies

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate

pip install -r requirements.txt
```

### 4. Run the app

```bash
streamlit run app.py
```

---

## Configuration

To adapt this for a different restaurant, edit the top of `vector.py`:

```python
RESTAURANT_NAME = "Your Restaurant Name"
CSV_FILENAME = "your_reviews.csv"
```

Then replace the PDF and image files with your restaurant's documents and delete the `chrome_langchain_db/` folder so the vector database rebuilds on next run.

## Architecture

```
User question
     │
     ▼
Embedding model (all-minilm)
     │
     ▼
Chroma vector DB ──► Top 5 relevant chunks
     │
     ▼
LangChain prompt (chunks + question)
     │
     ▼
Llama 3.2 (local via Ollama)
     │
     ▼
Answer (grounded in documents only)
```

---

## Notes

- The vector database is built on first run and cached in `chrome_langchain_db/`. Delete this folder to force a rebuild.
- All inference runs locally — no OpenAI API key or internet connection needed after setup.
- The sample data is synthetic and used for demonstration purposes only.
